# Run Definition of Done v02

A run is done only when all of the following are true:

1. The task reached a stable terminal state.
2. All required validations were completed.
3. Required guardian gates were completed.
4. Required governance approval was obtained when applicable.
5. Required verification evidence exists when applicable.
6. Outputs were written to the correct queue, report, or artifact path.
7. Trace record exists and is complete.
8. Follow-up state is explicit.
9. Memory writeback, if any, obeys promotion policy.
