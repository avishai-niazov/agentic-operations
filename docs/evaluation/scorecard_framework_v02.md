# Scorecard Framework v02

## Purpose

Score outcomes with enough structure to support promotion, rollback, or revision.

## Core dimensions

- task completion
- constitutional alignment
- technical safety
- pedagogical quality
- brand distinctiveness
- search value
- monetization safety
- operator effort saved
- repeatability

## Score classes

- `strong`
- `acceptable`
- `weak`
- `reject`

## Promotion rule

Only `strong` and selected `acceptable` outcomes may produce approved patterns.
