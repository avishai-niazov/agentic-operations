# Agentic Runtime Core v02

## Objective

Define the operational core for a governed multi-agent system that manages Planerium's daily work without sacrificing identity or educational quality.

## Core model

The runtime is composed of four layers:

1. Constitutional layer
2. Harness layer
3. Specialist agents
4. Memory and evaluation layer

## Why this model

Generic agent systems optimize for throughput. Planerium requires a system that optimizes for:
- trust
- distinctiveness
- teacher usefulness
- pedagogical integrity
- operational reliability
- controlled growth

## Core runtime claims

- The harness is the only orchestration authority.
- Agents are narrow and replaceable.
- Agents communicate through typed artifacts and queues.
- Evaluation determines learning promotion.
- Guardian agents exist to prevent drift, not to slow everything down.

## Core control loop

Signal → classify → route → execute → validate → decide → writeback → evaluate → learn or regress

## Runtime constraints

- No content publish path can bypass brand and pedagogy gates.
- No risky write can bypass governance.
- No agent can change its own charter.
- Market intelligence cannot be used to rewrite Planerium voice.
- Self-healing must preserve auditability.
